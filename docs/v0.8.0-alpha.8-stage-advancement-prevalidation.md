# v0.8.0-alpha.8 Stage Advancement Pre-validation

## Positioning

This package advances the uploaded real project from `v0.8.0-alpha.6` source-level closure to `v0.8.0-alpha.8` stage-advancement pre-validation.

It follows the prior implementation plan and does not redesign the workflow. Runtime validation remains deferred.

## Implemented scope

### alpha.7 source integrity repair

- Fixed `core/trace_backfill_service.py` top-level indentation corruption.
- Fixed the broken newline literal in `_input_for_trace()`.
- Synchronized version metadata to `0.8.0-alpha.8`.
- Updated stale package and delivery manifests.

### alpha.8 stage advancement closure

- Added pure-read `trace_backfill_gap` GateRule.
- Added StageBlocker contract support for `trace_backfill_gap`.
- Added resolution operations:
  - `trace_to_eval_case`
  - `create_trace_backfill_dataset`
- Added concrete StageResolutionOperation API bindings:
  - `POST /sessions/{session_id}/traces/{trace_id}/to-eval-case`
  - `POST /sessions/{session_id}/traces/to-eval-dataset`
- Added Stage 3 readiness metadata for trace backfill summary.
- Added final governance visibility for trace backfill blockers.
- Updated Gate Panel to show trace backfill blockers and operation hints.

## Stage advancement semantics

Stage 3 now treats failed/parser/safety traces as part of the regression-readiness closure:

1. Eligible traces must be converted into EvalCase records.
2. Trace-backfilled EvalCase records must be grouped into a `production_trace` EvalDataset.
3. Gate evaluation remains pure-read and does not create EvalCases or datasets.
4. Concrete execution remains in existing trace API endpoints.

## Deferred validation boundary

The following were not run:

- pytest
- import smoke
- FastAPI startup
- Streamlit startup
- Docker build
- live LLM
- live Search

These checks should be run later in the unified validation phase after all source-level functionality is frozen.

## Not included

- v0.9 Workspace / Project.
- RBAC.
- Claim-Evidence Graph.
- Report Center approval/publish workflow.
- Production CORS/auth/rate-limit hardening.
- Provider-native structured output replacement.
