# v0.8.0-alpha.1 Eval Dataset & Experiment Foundation

Status: source-level implementation patch; runtime validation deferred by instruction.

## Scope

This release upgrades the existing session-scoped EvalCase / EvalRun system into
a reusable evaluation foundation:

1. `EvalDataset`
2. `EvalAggregateMetrics`
3. `EvalExperiment`
4. Dataset service
5. Experiment service
6. Metrics and baseline comparison services
7. Dataset / experiment API routers
8. Storage side tables and migration entries
9. Streamlit panel additions
10. Contract tests for later unified validation

## Alignment with v0.7-alpha.4

This release does not bypass `StageAdvancementDecision`.

EvalExperiment results are stored as metrics, comparison summaries, audit events,
and eval traces. They do **not** create StageGate blockers in alpha.1.

## Deferred to v0.8-alpha.2

- `core/gates/rules/eval_regression.py`
- Regression Gate blocker generation
- StageAdvancementDecision consumption of experiment regression results
- Frontend gate panel regression operations
