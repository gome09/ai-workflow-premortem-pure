### 本机环境问题（非代码缺陷，未提交任何改动）

第一轮 pytest 出现 68 个 `PermissionError: [WinError 5]`，根因是 `C:\Users\embar\AppData\Local\Temp\pytest-of-embar` 目录 ACL 损坏（takeown/icacls/rd 均拒绝访问，疑似残留句柄或损坏 ACL）。处置：将 `TMP/TEMP` 重定向到 `.upgrade/tmp/pytest-temp` 后重跑全绿。**建议用户重启后手动删除该目录**；若日常本机跑测试再遇同类错误，可用相同的 TMP 重定向绕过。

## Phase B：远端 CI

### 基线（修复前）

修复前最新 run [29621280076](https://github.com/gome09/ai-workflow-premortem-pure/actions/runs/29621280076)（2026-07-17）conclusion=failure：
- `lint-and-unit-tests` 红：doc-check 在 Linux 报 9 处"反引号路径不存在"（`tests/...`、`.upgrade/reports/...` 等省略号占位路径）。**本地 Windows 不报**——Windows 文件系统忽略路径结尾点号使 `exists()` 误判通过，属平台行为差异（计划预判的"本地过远端不过"类别）。
- `docker-full-integration` 红（non-blocking）：api 容器崩溃循环 `SettingsError: error getting value for field "deepseek_api_key" from SecretsSettingsSource` + grafana `Permission denied` 读 `/run/secrets/grafana_password`。根因：`gen_secrets.sh` 对 secrets 文件 `chmod 600`，CI runner 上属主为 runner 用户，容器内非 root 用户（api 的 appuser、grafana）无权读挂载的 `/run/secrets/*`。本地 Windows Docker Desktop 挂载不保留 POSIX 权限故不复现。

### 修复（commit 5b4003f，1 轮通过，未达 3 轮上限）

| 文件 | 改动 | 性质 |
|---|---|---|
| `scripts/doc_consistency_check.py` | 规则 3 跳过以 `..` 结尾的省略号占位路径（如 `tests/...`、`alembic/versions/V005_..`），双平台行为一致 | **生产代码（工具脚本）** |
| `.github/workflows/ci.yml` | docker-full-integration 生成 secrets 后 `chmod 644 secrets/*`（CI 一次性随机值，放宽仅限 runner） | CI 配置 |

修复后本地复验：doc-check 0 违规、version-check OK、ruff clean、全量 mypy 仍 0（注：单独 `mypy scripts/doc_consistency_check.py` 有 2 个既有 error，但该脚本不在 mypy files 白名单内，非本次引入，全量 `uv run mypy` 不受影响）。
