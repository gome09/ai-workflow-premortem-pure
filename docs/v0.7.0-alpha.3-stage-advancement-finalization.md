# v0.7.0-alpha.3 Stage Advancement Finalization

## Stage positioning

This package advances the project from `v0.7.0-alpha.2 stage-advancement closure`
to `v0.7.0-alpha.3 stage-advancement finalization`.

The project is still a pre-validation alpha package. It is not a v0.8 Eval /
Red Team release and is not a v0.9 Governance Workbench release.

## Objective

Stabilize the stage-advancement backbone before adding v0.8/v0.9 features.

The main workflow remains:

```text
single_step runner
  -> stage executor
  -> StructuredOutputClient boundary
  -> Review Gate
  -> GateEngine
  -> PendingHumanAction resolution contract
  -> stage transition / report summary / trace surface
```

## In scope

1. Align package/version metadata.
2. Convert the first four core GateRules from compatibility wrappers to direct
   rule classes:
   - `missing_output`
   - `stale_dependency`
   - `action_state`
   - `parser_error`
3. Tighten the action-resolution contract around stale/conflict/replay/hash
   semantics.
4. Add structured-output provider boundary scaffolding without forcing a live
   provider integration.
5. Add stage-advancement summary to reports.
6. Add minimal gate and trace frontend components.
7. Keep all runtime validation deferred.

## Out of scope

1. pytest execution.
2. API / Streamlit / Docker startup validation.
3. Live DeepSeek/OpenAI-compatible provider validation.
4. Live Tavily/search validation.
5. EvalDataset / EvalExperiment / Regression Gate.
6. RedTeamCase and taxonomy mapping.
7. Workspace / RBAC / Claim-Evidence Graph / Report Center.

## Exit criteria before unified validation

Source-level criteria:

- release metadata is internally consistent;
- first-batch GateRules no longer call legacy collector functions;
- action-resolution statuses are documented and return clearer hash fields;
- structured-output provider interfaces exist;
- report JSON includes `stage_advancement_summary`;
- frontend has minimal `gate_panel.py` and `trace_panel.py`;
- download-link recovery is documented.

Runtime validation remains deferred until the later unified test pass.
