# v0.8.0-alpha.2 Regression Gate Integration

This source patch implements the next phase defined after v0.8.0-alpha.1. It does not redesign the project and does not run runtime validation.

## Scope

- Keep the deterministic single-step workflow and existing EvalDataset / EvalExperiment foundation.
- Add a pure-read Stage 3 `eval_regression` GateRule.
- Consume existing `EvalDataset`, `EvalExperiment`, and `comparison_summary` records.
- Block Stage 3 advancement when a gate-relevant dataset lacks a current experiment, lacks a comparison, or shows regression against baseline.
- Add concrete StageResolutionOperation metadata for `run_eval_experiment` and `compare_eval_experiment`.
- Expose regression gate status in reports and the Streamlit panels.

## Validation Boundary

Not run in this patch:

- pytest;
- import smoke tests;
- FastAPI startup;
- Streamlit startup;
- Docker build;
- live LLM calls;
- live Tavily/Search calls;
- E2E workflow execution.

Unified validation remains deferred until all planned v0.8 alpha functionality is complete.
