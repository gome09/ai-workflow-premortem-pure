# v0.8.0-alpha.3 Red Team Foundation

Status: `pre-validation-redteam-foundation`.

This patch follows the previously agreed stage-progression design. It does not redesign the project and does not run pytest, API startup, Streamlit startup, Docker, LLM, or Search validation.

## Scope

- Add `RedTeamCase` as a session-scoped v0.8 model.
- Add deterministic `core/redteam_service.py` generation from real project state:
  - high / critical `SafetyFinding`;
  - high / critical `FailureMode`;
  - high-risk `WorkflowNode` / oversight gaps.
- Add `RedTeamCase -> EvalCase` sync.
- Add `redteam_generated` EvalDataset creation.
- Add `redteam_coverage` Stage 3 GateRule.
- Add API routes under `/sessions/{session_id}/redteam/*`.
- Add minimal Streamlit Red Team coverage panel.
- Add report summary fields for Red Team coverage.
- Add storage side-table sync for `redteam_cases`.
- Add alpha3 contract tests as source files only.

## Stage progression contract

Stage 3 now has the following intended order:

1. stage output and stale dependency checks;
2. parser and human-action blockers;
3. Stage 3 eval coverage / failure checks;
4. Red Team coverage check;
5. Eval Regression Gate check.

The new `redteam_coverage` rule is pure-read. It does not generate, approve, sync, run, or compare anything during gate evaluation.

## Deferred validation

The package is intentionally not validated at runtime in this handoff. Unified validation should happen after the remaining v0.8 work is complete:

- taxonomy mapping;
- EvalJudgment / HumanCalibration;
- trace-to-eval regression backfill;
- v0.8 report and dashboard closure.
